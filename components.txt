Guide de Développement : L'Usage des Props
Pour Amin et toi, voici comment structurer vos composants React pour qu'ils soient dynamiques et "Data-Compatible".

A. Composant TodoItem
Ce composant doit réagir visuellement à la priorité de la tâche.

Props attendues : task (string), priority (string), isCompleted (boolean), onToggle (fonction), onDelete (fonction).

Expert Tip : Utilise un objet de mapping pour les couleurs (ex: { high: 'bg-red-500', medium: 'bg-yellow-500' }).

B. Composant ContactCard
Il doit être capable de gérer les données manquantes (comme l'email optionnel).

Props attendues : firstname, surname, phonenumber, email, onEdit, onDelete.

Expert Tip : Utilise le Conditional Rendering : {email && <p>{email}</p>} pour éviter d'afficher une ligne vide si l'email n'existe pas.

C. Composant NoteCard
L'enjeu ici est l'alignement visuel (layout stability).

Props attendues : title, content, createdAt, onDelete.

Expert Tip : Implémente une fonction de "Truncate" pour le content afin qu'il ne dépasse jamais une certaine longueur dans la vue d'ensemble.

Plan de Travail Détaillé
Dashboard Layout : Créer le conteneur principal en display: grid avec 3 colonnes de 1fr.


Glassmorphism Modals : Créer un composant Modal réutilisable avec backdrop-filter: blur(10px).


Atomic Design : Développer les composants TodoItem, ContactCard et NoteCard en utilisant strictement les Props définies ci-dessus.
